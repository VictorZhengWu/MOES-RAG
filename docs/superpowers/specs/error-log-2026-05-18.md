## M6 用户端错误

### E001: 语言切换 URL 双重 locale
**现象**：切换语言后 URL 变为 `/zh/en/chat` 导致 404
**根因**：`usePathname()` 返回含 locale 的完整路径 `/en/chat`，但代码认为是去掉前缀的路径，用 `/${newLocale}${pathname}` 拼接成 `/zh/en/chat`
**修复**：改用分割路径段、替换第一个 segment 的方式
**文件**：`language-switcher.tsx`

### E002: 流式输出引用编号不显示
**现象**：AI 回答完成后消息底部没有 `[1] DNV Rules...` 引用编号
**根因**：Mock Server 的 `mock_stream_chunks()` 只返回文字 token，不含 citations；citations 仅在 `mock_chat_response()` 中
**修复**：在 `[DONE]` 前插入一个包含 citations 的 chunk
**文件**：`mock-server/src/mock_server/data.py`

### E003: 停止按钮无效
**现象**：AI 输出中按停止按钮，AI 仍然持续生成
**根因**：AbortController 创建了但 `signal` 没有传给 `fetch()`
**修复**：`streamChat()` 接受 `AbortSignal` 参数，传入 fetch；hook 创建 controller 后传入
**文件**：`api/chat.ts`, `hooks/use-chat-stream.ts`

### E004: 停止按钮对 Follow-up 建议无效
**现象**：点击 AI 推荐的后续问题，生成中按停止无效；手动输入的问题可以停止
**根因**：`useChatStream()` 在 ChatInput 和 MessageList 中各实例化一次，各有独立的 `useRef(AbortController)`。ChatInput 的停止按钮只能中止自己实例的流
**修复**：将 `abortRef` 从组件级 `useRef` 改为模块级 `sharedAbortRef`，所有实例共享同一控制器
**文件**：`hooks/use-chat-stream.ts`

### E005: LanguageSwitcher button-in-button hydration 错误
**现象**：浏览器控制台报 `<button> cannot be a descendant of <button>`，页面功能失效
**根因**：shadcn/ui (Base UI) 的 `DropdownMenuTrigger` 渲染 `<button>`，内部又放了 `<Button>` 组件（也是 `<button>`）
**修复**：移除内部 `<Button>`，将 className/onClick 直接放在 `DropdownMenuTrigger` 上
**文件**：`language-switcher.tsx`

### E006: 新增 i18n key 后韩文界面全部英文
**现象**：切换到韩文后 UI 仍显示英文
**根因**：`ko.json` 写入失败（需要先 Read 才能 Write），内容仍是英文
**修复**：重新读取 `ko.json` 后写入完整韩文翻译
**文件**：`messages/ko.json`

### E007: i18n key 缺失导致整个应用崩溃
**现象**：新增 `sidebar.projects` key 只加了 `en.json`，其他 4 个 locale 文件没有。用户打开页面 → MISSING_MESSAGE 异常 → React 树崩溃 → 全部功能失效
**根因**：next-intl 的 `t('missing.key')` 会抛出异常，不是 fallback 到空字符串。ConversationSidebar 在所有页面渲染，一崩全崩
**教训**：新增 i18n key 必须同时更新全部 6 个 locale 文件
**文件**：所有 `messages/*.json`

### E008: 引用面板打开后背景模糊，内容被遮住
**现象**：点击引用编号 → 中间对话区域变模糊，文字被浮动面板遮挡
**根因**：shadcn/ui 的 `Sheet` 组件渲染 `SheetOverlay`（`backdrop-blur-xs`），模态遮罩导致背景模糊
**修复**：不用 Sheet 组件，改为纯 CSS 滑出面板（`fixed` → 后改为第三列布局）
**文件**：`citation-panel.tsx`, `app-layout.tsx`

### E009: TooltipTrigger / DropdownMenuTrigger button-in-button
**现象**：侧边栏图标条和会话 "..." 菜单出现嵌套 button 警告
**根因**：同 E005，Base UI 的 `TooltipTrigger` 和 `MenuTrigger` 渲染 `<button>`，内部不能再放 `<button>`
**修复**：将所有 Trigger 内部的 `<button>` 改为直接挂在 Trigger 上
**文件**：`collapsed-sidebar.tsx`, `conversation-item.tsx`

### E010: Share 按钮被跳转窄条遮挡
**现象**：鼠标移到 header 右侧的 Share 图标 → 跳转窄条的列表框弹出，无法点击 Share
**根因**：跳转窄条 `fixed top-0`，hit area 覆盖了整个视口高度，遮挡了 header 区域
**修复**：跳转窄条 `top: 44px`（header 高度），不覆盖 header 区域
**文件**：`jump-navigation.tsx`

### E011: 跳转窄条 hit area 覆盖按钮
**现象**：鼠标在最上和最下横线之间无法稳定触发列表框
**根因**：单独的 `hit area` div 覆盖在按钮上方拦截了指针事件
**修复**：去掉独立 hit area，把 `onMouseEnter/onMouseLeave` 直接挂在线条容器和统一盒子上
**文件**：`jump-navigation.tsx`

### E012: `top:50%` + `translateY(-50%)` + `marginTop` 冲突
**现象**：hit 区域错位，和视觉线条不重合
**根因**：同时使用三种居中方式导致双重偏移
**修复**：去掉 `marginTop`，仅用 `top:50%` + `translateY(-50%)`
**文件**：`jump-navigation.tsx`

### E013: Suggestion 按钮文字溢出
**现象**：建议问题按钮文字出框，不换行
**根因**：Button 无 `whitespace-normal break-words`
**修复**：添加 `whitespace-normal break-words min-w-0`
**文件**：`empty-state.tsx`

### E014: 登录后跳转丢失对话内容
**现象**：退出登录 → 重新登录 → 之前的对话内容消失
**根因**：登录后 `router.push('/locale/chat')` 总是跳到新对话，清除当前消息。且 `/chat/[id]` 页每次 mount 都 `clearMessages()` + 重新 fetch Mock 数据
**修复**：登录/注册传 `redirect` 参数回到原页面；`/chat/[id]` 页改为只在 ID 变化时重新加载
**文件**：`conversation-sidebar.tsx`, `login-form.tsx`, `chat/[id]/page.tsx`

### E015: 对话框 DialogDescription 嵌套 `<p>` 和 `<div>`
**现象**：上传无效文件后弹窗 → hydration error → 页面所有按钮无法点击
**根因**：`DialogDescription` 渲染 `<p>` 标签，内部不能放 `<p>` 或 `<div>`（HTML 规范）
**修复**：将嵌套内容改为普通 `<div>`，不用 `DialogDescription`
**文件**：`documents/page.tsx`

## M7 管理后台错误

### E016: M7 所有页面 404
**现象**：启动 M7 后 `http://localhost:3000` 全部 404
**根因**：Phase C-1 只建了根布局和 middleware，没建 `[locale]/layout.tsx` 和 `[locale]/page.tsx`。next-intl 中间件路由正常但无渲染出口
**修复**：创建 `[locale]/layout.tsx`（NextIntlClientProvider）+ `[locale]/page.tsx`（重定向到 /admin）
**文件**：`[locale]/layout.tsx`, `[locale]/page.tsx`

### E017: M7 页面重定向到旧占位页
**现象**：打开 M7 显示 "MO Expert Admin — Phase C-1 complete" 而不是 Admin 登录页
**根因**：`[locale]/page.tsx` 还是旧占位文字，没有重定向到 `/admin`
**修复**：改为 `redirect('/{locale}/admin')`
**文件**：`[locale]/page.tsx`

### E018: 语言切换后 KG/LLM/Users/Monitoring 页面仍显示英文
**现象**：切换语言后标题和部分内容仍是英文
**根因**：这些页面的标题和副标题使用了硬编码英文，没有用 `t()` i18n 调用
**修复**：为每个页面的标题、副标题、表单标签添加 `t()` 调用；添加 `useTranslations` import
**文件**：`knowledge-graph/page.tsx`, `llm-config/page.tsx`, `users/page.tsx`, `monitoring/page.tsx`

### E019: sidebar.logIn key 缺失
**现象**：控制台报 `MISSING_MESSAGE: Could not resolve sidebar.logIn`
**根因**：admin-sidebar 使用了 `t('sidebar.logIn')` 但 M7 的 en.json 中没有 `sidebar.logIn` key
**修复**：改为直接硬编码 `Log in`（admin 专用文字）
**文件**：`admin-sidebar.tsx`

### E020: 用户表后三列右对齐
**现象**：Queries、API Keys、Actions 列右对齐
**根因**：表头和单元格使用了 `text-right` 类
**修复**：改为 `text-left`，Table 组件加 `[&_td]:text-left [&_th]:text-left`
**文件**：`users/page.tsx`, `documents/page.tsx`

### E021: 文档表排序和列宽无效
**现象**：点击表头无排序，拖动列边缘无反应
**根因**：
- 排序：添加了 `handleSort`/`sortIcon` 但表未连线 `sorted` 数组
- 列宽：`resize-x` 在 `display: table-cell` 上无效
**修复**：用内嵌 `<div className="inline-block resize-x overflow-auto">` 实现列宽调整；确保 `filtered` 数组包含 `.sort()` 结果
**文件**：`documents/page.tsx`, `users/page.tsx`

### E022: 马来文导致语言切换 404
**现象**：切换到马来文后出现 `/en/ms/settings` 双重 locale URL
**根因**：`language-switcher.tsx` 和 `settings/page.tsx` 中 locale 验证数组 `['en','zh','ko','ja','no']` 没有包含 `ms`；遇到 `ms` 时无法识别为 locale，导致 prepend 而非 replace
**修复**：全部 5 个 locale 验证数组都加上 `'ms'`
**文件**：`language-switcher.tsx`(M6+M7), `general-tab.tsx`, `settings/page.tsx`(M7)

## 通用 / 基础设施错误

### E023: 端口 8000 被系统进程占用
**现象**：Mock Server 无法绑定 8000 端口
**根因**：Windows 系统进程 PID 7952 占用 8000，且 `taskkill` 无权终止
**修复**：Mock Server 改用 8002/8003；M6/M7 创建 `.env.example` 指定 API URL；Playwright config 同步更新
**文件**：`.env.example`(M6+M7), `playwright.config.ts`

### E024: create-next-app 无法在已有目录运行
**现象**：M7 脚手架创建失败，提示目录中已有文件
**根因**：初始项目骨架阶段已创建 `m7-admin-portal/src/` 和 `tests/` 空目录
**修复**：`rm -rf m7-admin-portal` 后重新运行
**教训**：创建 app 之前确认目标目录为空

### E025: shadcn toast 已废弃
**现象**：`npx shadcn@latest add toast` 报错 "toast is deprecated. Use sonner instead"
**根因**：shadcn v4 (2026) 废弃 toast 组件
**修复**：改用 sonner 组件
**文件**：`components/ui/sonner.tsx`

### E026: next-intl 4.x 需要 Plugin
**现象**：页面报 `Couldn't find next-intl config file`
**根因**：next-intl 4.x App Router 模式需要在 `next.config.ts` 注册 `createNextIntlPlugin()`
**修复**：`export default withNextIntl(nextConfig)`
**文件**：`next.config.ts`(M6+M7)

### E027: Playwright webServer 环境变量不传递
**现象**：Playwright 测试中 API 调用报 `Failed to fetch`
**根因**：`reuseExistingServer: true` 时重用旧服务器，旧服务器没有 `NEXT_PUBLIC_API_URL` 环境变量
**修复**：Playwright config 设置 `env` 字段，确保 webServer 携带正确的 env
**文件**：`playwright.config.ts`

## 经验教训

1. **i18n key 新增 = 全文件同步**：新增 key 必须同时更新 all locale JSON，否则应用崩溃
2. **Base UI 不支持 asChild + 不能嵌套 button**：`DropdownMenuTrigger`/`TooltipTrigger` 本身就渲染 `<button>`
3. **DialogDescription 是 `<p>`**：不能放 block 元素
4. **useRef 是组件级**：多个组件实例各自独立，需要共享用模块级
5. **CSS resize 对 table-cell 无效**：需用内嵌 block 元素
6. **Mock Server 数据格式必须和 UI 设计一致**：否则前端加载后显示 default 状态
7. **语言 locale 数组是散落的**：增加语言时需检查所有 `['en','zh',...]` 数组
