## 代码审查发现并已修复的问题

### F001: logger.exception 在 health/init/close 中打印完整堆栈

**现象**：Meilisearch 不可用时，Demo 输出满屏红字堆栈（~80 行）
**根因**：`health_check()` 使用 `logger.exception()`（打印完整 traceback），`manager.py` 使用 `logger.error()` 打印异常对象
**修复**：全部改为 `logger.warning()`，只输出一行提示。系统设计就是允许部分后端 unavailable 的
**文件**：`chromadb_store.py`, `meilisearch_index.py`, `sqlite_db.py`, `manager.py`

### F002: src/ 目录布局在 Windows Python 3.13 上 editable install 失败

**现象**：`pip install -e m2-storage/` 后 `import m2_storage` 报 ModuleNotFoundError
**根因**：`[tool.setuptools.package-dir] "m2_storage" = "src"` 在 editable 模式下不工作
**修复**：将 `src/` 重命名为 `m2_storage/`，删除 `package-dir` 和 `packages.find` 配置
**文件**：`m2-storage/m2_storage/`, `pyproject.toml`

### F003: Python 源文件不含 UTF-8 编码声明导致 GBK 解码失败

**现象**：`import m2_storage` 报 `UnicodeDecodeError: 'gbk' codec can't decode byte 0x94`
**根因**：中文 Windows 默认编码为 GBK，源文件中的 em dash（—）无法被 GBK 解码
**修复**：给含非 ASCII 字符的源文件添加 `# -*- coding: utf-8 -*-` 头
**文件**：`__init__.py`, `test_config.py`, `test_meilisearch.py`, `meilisearch_index.py`

### F004: requirements.txt 缺少 marine-rag-contracts

**现象**：`pip install -r requirements.txt` 后 import 失败
**根因**：所有后端都从 `contracts.storage` 和 `contracts.document` 导入，但 contracts 不在 requirements.txt 中
**修复**：添加注释说明需 `pip install -e ../contracts/`
**文件**：`requirements.txt`

### F005: deployment_mode 无校验

**现象**：拼写错误的值（如 "persnal"）被静默接受
**修复**：在 `StorageConfig.from_dict()` 中添加 `personal/enterprise/saas` 三值校验，非法值抛 ValueError
**文件**：`config.py`

### F006: doc_id 派生用 .replace(".pdf", "") 脆弱

**现象**：`.PDF`、`.DOCX` 或无扩展名的文件产生不一致的 doc_id
**修复**：改用 `Path(source_filename).stem`
**文件**：`chromadb_store.py`, `meilisearch_index.py`

### F007: _is_safe_key(".") 返回 True

**现象**：`"."` 作为 key 会解析到存储根目录
**修复**：新增 `if normalized == ".": return False`
**文件**：`local_fs.py`

### F008: Meilisearch delete() 直接拼接 filter 字符串

**现象**：`delete()` 用 f-string 构建 filter，`search()` 用 `_build_meili_filter()`，不一致且有注入风险
**修复**：`delete()` 改为调用 `_build_meili_filter({"doc_id": doc_id})`
**文件**：`meilisearch_index.py`

## 经验教训

1. **Windows GBK 编码陷阱**：中文 Windows 默认编码非 UTF-8，含非 ASCII 字符的 Python 源文件必须加 `# -*- coding: utf-8 -*-`
2. **editable install 不可靠**：`setuptools package-dir` 在 editable 模式下行为不一致，优先用常规目录布局
3. **健康检查不应打印堆栈**：`logger.exception()` 适合未预期错误，周期性健康检查失败是预期行为，应用 `logger.warning()`
4. **`.replace(".pdf", "")` 是反模式**：用 `Path.stem` 处理任意扩展名
5. **所有 f-string 拼接 filter 都应有统一构建函数**：`_build_meili_filter()` 已存在但 `delete()` 没用它 — 代码审查比测试更容易发现这类不一致
