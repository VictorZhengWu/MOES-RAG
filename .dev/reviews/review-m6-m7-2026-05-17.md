## 二、代码质量问题

### 2.1 全局性问题

**1. 硬编码 API 地址**

M6 和 M7 的 `src/lib/api/client.ts` 中 `BASE_URL` 默认值是硬编码的 `http://127.0.0.1:8000`。用户机器上 8000 端口被占用，每次都需要 `set NEXT_PUBLIC_API_URL=http://127.0.0.1:8002`。

建议：在 `deploy/` 中创建 `.env.development` 文件，指定默认值 8002。或者创建 `.env.example` 说明配置方法。

**2. Mock Server 假数据不适配新 UI**

Mock Server 的 LLM backend 数据还是旧格式（无 purpose 字段），导致 LLM 配置页加载后全部默认。

建议：重新对齐 Mock Server 数据与 M7 7-box 设计。

**3. M6 和 M7 共享代码重复**

以下代码在 M6 和 M7 中完全重复：

- `src/lib/api/client.ts`（API 客户端）
- `src/components/layout/language-switcher.tsx`（语言切换器）
- i18n 中间件和配置
- Playwright config 模式

建议：抽到共享 `packages/shared/` 目录。

### 2.2 M6 特定问题

- `use-chat-stream.ts` 没有处理网络断连重试
- 空状态 5 个建议问题是硬编码在 `en.json` 中的，不能动态生成
- Share 链接是假的（`https://mo-expert.com/share/xxx`）

### 2.3 M7 特定问题

- 文档上传的 `uploadDocument` 调用只传了 `file_name`，没有传文件内容（Phase 1 预期）
- 用户管理用 `createUser` 但无 `updateUser` API（编辑使用占位逻辑）
- 监控页面日志是硬编码的 6 条假日志

## 三、结合最新 RAG 技术的功能缺口

基于 2025-2026 年 RAG 领域的最新技术趋势，以下功能尚未在系统中体现：

### 3.1 检索增强层（影响 M6 用户体验）

| 功能 | 现在的状态 | 建议 |
|------|---------|------|
| **答案来源混合标识** | 引用仅标注 "Source: DNV Pt.4 Ch.3" | 应区分来源类型：📖 向量检索 / 🧠 KG 推理 / 🌐 Web Search；每种用不同颜色标签 |
| **检索结果解释** | 无 | 在回答中或引用面板中展示 "系统检索了 3 份规范、匹配到 15 个段落、重排后取前 5" |
| **HyDE / Query Rewriting 可视化** | 无 | 高级用户想看到 "系统实际用的查询"——在引用面板中加一行小字 "系统将此问题改写为: ..." |
| **Confidence Score** | 无 | 每条回答显示可信度评分（基于检索到的文档与回答的匹配度） |
| **答案反馈闭环** | 无 | 用户点 👍/👎 → 反馈到检索/生成流水线，持续优化 |
| **Follow-up 建议** | 无 | 回答末尾自动生成 3 个追问建议（如 ChatGPT 那样） |

### 3.2 Agent 层（影响 M6 高级功能）

| 功能 | 现在的状态 | 建议 |
|------|---------|------|
| **Deep Research Agent** | 按钮占位 | 多步检索→阅读→再检索→合成报告。UI 需展示中间步骤 |
| **多文档对比** | 无 | "DNV vs ABS 对 LNG 货舱的要求对比"→系统并行检索两套规范→分栏对比结果 |
| **图表/公式识别** | 无 | M1 VLM 解析图纸 → M6 可以在回答中展示原图，高亮标注 |
| **Code-aware RAG** | 无 | 如果用户问计算公式，回答中应显示公式原文（LaTeX 渲染），不是纯文字 |

### 3.3 知识管理增强（影响 M7）

| 功能 | 现在的状态 | 建议 |
|------|---------|------|
| **KG 手动修正** | ❌ 未实现 | 设计规范既有要求 |
| **文档去重检测** | ❌ | 上传时检查是否有内容高度重叠的文档已存在 |
| **规范版本对比** | ❌ | "DNV Pt.4 2024 vs 2025 变更对比" 高亮差异 |
| **RAG 评估面板** | ❌ | 管理员上传测试问题集 → 系统自动跑 → 展示 Recall@K / Precision / 幻觉率 |
| **检索质量仪表板** | ❌ | 监控页面加一个"检索质量"标签：平均召回率、热门搜索词、零结果搜索词 |
| **System Prompt 模板管理** | ❌ | M7 中管理 M5 使用的 System Prompt 模板（不同专业可用不同提示词） |

### 3.4 多模态

| 功能 | 现在的状态 | 建议 |
|------|---------|------|
| **图纸/图片上传** | M6 有附件按钮但仅占位 | 上传图纸 → VLM 生成文字描述 → 存入向量库 |
| **回答中的图片展示** | 无 | 当 RAG 结果包含图表时，在回答中内嵌展示原始图片 |

## 四、优先级建议

### 🔴 高优先级（应当立即处理）

1. **M7 管理员登录保护** — 安全漏洞，任何人都可访问管理后台
2. **KG 手动修正** — 设计规范要求，目前只读
3. **API 地址配置化** — 解决每次手动 `set NEXT_PUBLIC_API_URL` 的问题
4. **对齐 Mock Server 数据** — 让 LLM 配置页能用

5. **答案反馈 (👍/👎)** — 为标准 RAG 功能
6. **答案来源混合标识** — 提升用户信任度
7. **Follow-up 建议** — 提升用户粘性
8. **对话内容复制按钮** — 基础 UX

11. **多文档对比 UI** — 依赖 M5
12. **规范版本对比** — 依赖 M4 KG
