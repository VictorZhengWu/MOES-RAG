目前有很多先进的RAG技术，如下：
一、检索层面的增强
1. 混合检索（Hybrid Search）
将稠密向量检索（Dense） 和稀疏关键词检索（Sparse，如 BM25） 结合

利用 Dense 捕捉语义，Sparse 保证精确关键词匹配

典型实现：Weaviate、Pinecone、Elasticsearch 的混合检索模式

2. 多路召回与融合排序（Multi-Route Retrieval + Fusion）
同时走多条检索路径（如向量库 + 知识图谱 + 全文搜索 + API）

用融合排序（如 RRF：Reciprocal Rank Fusion）合并结果

可结合不同粒度的索引（句子级、段落级、文档级）

3. 知识图谱增强 RAG（Graph RAG）
将语料构建为知识图谱（实体-关系-实体）

检索时查询相关子图，利用图结构进行多跳推理

典型方案：Microsoft GraphRAG、NebulaGraph + LLM

4. 层级化 / 导航式 RAG（Hierarchical / Navigational RAG）
将文档组织为树形或层级结构（章节→段落→句子）

检索时从粗到细“导航式”定位，避免在大海捞针

适合超长文档、书籍、手册等场景

5. 时间感知 / 上下文窗口检索
引入时间戳、版本号等元数据过滤

适合法律、金融等对时效性要求高的场景

二、查询理解与重写
6. 查询重写（Query Rewriting）
用 LLM 将模糊、简短的用户问题改写成更清晰、更适合检索的形式

可生成多个改写版本分别检索

7. 查询分解（Query Decomposition / Sub-Question Splitting）
将复杂问题拆解为多个子问题

分别检索、回答，再做汇总整合

典型：Least-to-Most Prompting、IRCoT

8. 假想文档嵌入（HyDE）
先让 LLM 生成一个虚构的“理想答案”

用这个假想答案的 embedding 去检索真实文档

解决 query-document 语义 gap 问题

9. Step-Back Prompting
引导模型先抽象出更高层次的概念问题

用概念层面问题辅助检索和推理

三、索引与文档处理增强
10. 语义分块（Semantic Chunking）
按语义边界而非固定长度切分文档

用句子嵌入相似度断点、或 LLM 动态判断分块位置

11. 多粒度索引（Multi-Granularity Indexing）
同时建立句子级、段落级、文档级的索引

检索时按问题需求动态选择合适的粒度

12. 主动信息提取式索引（Proposition / Factoid Indexing）
不在文档块层面索引，而是用 LLM 事先把文档抽取为原子事实、命题

索引的是“知识点”而非原文片段

代表：Dense X Retrieval 概念、FactoidRAG

13. 摘要化索引（Summary-Indexed RAG）
对每篇文档先生成摘要、提取主题关键词

检索时先匹配摘要，缩小范围后再深入原文

四、生成与检索的深度融合
14. 自反思 / 递归 RAG（Self-RAG / Recursive RAG）
生成过程中持续评估是否需要检索、检索的结果是否有用

可多次检索，每一步对上一轮知识进行验证和修正

有专门的“反思标记”控制行为

15. 检索-生成交替迭代（ITER-RETGEN）
先检索一次→生成答案→用生成内容再次检索→再次生成

形成闭环迭代，逐步逼近正确答案

16. 验证与矫正型 RAG（CRAG / Corrective RAG）
对检索到的文档先做一个“可信度评估”

低质量的文档直接丢弃或触发更高精度的检索（如 Web 搜索 fallback）

代表：Corrective RAG

17. 逐步推理 + 检索（Interleaved Reasoning & Retrieval）
将“思维链”（Chain-of-Thought）与检索交替进行

每一步推理时按需去查外部知识

代表：IRCoT、ReAct 模式

五、Agent 化与多步编排
18. Agentic RAG
将 RAG 整体看作一个自主 Agent 的行为

系统能规划步骤、选择工具、判断何时检索、何时计算、何时调用 API

常结合 LangGraph、AutoGen、CrewAI 等框架

19. 工具增强 RAG（Tool-Augmented RAG）
除了检索，还能调用计算器、SQL 数据库、第三方 API

根据问题类型动态选择该用检索还是其他工具

20. 多智能体协作 RAG
多个专业 Agent 分工：查询重写 Agent、检索 Agent、验证 Agent、摘要 Agent 等

协作完成复杂的知识密集任务

六、多模态与跨模态 RAG
21. 多模态 RAG（Multimodal RAG）
同时支持文本、图片、音频、视频的检索和生成

使用多模态 embedding（如 CLIP、ImageBind）对齐不同模态

可图搜文、文搜图、视频问答等

22. 跨模态翻译检索
将一种模态的查询“翻译”成另一种模态后再检索

例如用语音提问，转为文本后检索，最后生成语音回答

七、适应性与专业化 RAG
23. 自适应 RAG（Adaptive RAG）
根据问题复杂度自动选择检索策略

简单问题可能不走检索，复杂问题走完整多步检索流程

24. 个性化 / 用户建模 RAG
维护用户画像、记忆、偏好

检索时融入用户个性化信息，提高回答针对性

25. 领域特化 RAG
针对代码、法律、医疗、金融等特定领域做索引、检索、评估的专门优化

通常结合领域实体识别、专业术语归一化等

八、评估与质量保障
26. RAG 专用评估体系
基于 RAGAS 等框架，从忠实度、答案相关性、上下文相关性、准确率等多维评估

引入 Critic / 评分模型作为在线质量裁判

27. 幻觉检测与缓解
检索后增加“事实验证”步骤

生成后使用 NLI 模型或 LLM 自检是否有不支持的内容

1	层级化导航式 RAG (#4)	船级社规范天生的树形结构（Pt→Ch→Section→Clause）。当前检索搜全部 50 页，导航式先定位到 Ch.3 再搜该章节内——精度提升不是 10%，是 2-3 倍。而且 M1 已经解析出了 chapter_section，元数据早就在了，缺的只是分层索引
