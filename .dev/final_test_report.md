## 关键问题清单

### 🔴 P0 - 严重问题 (阻塞发布)

#### 问题 #1: M8 API Gateway 语法错误
**文件**: `m8-api-gateway/m8_gateway/routes/auth.py:752`  
**错误**: 未关闭的括号导致语法错误  
```python
# 错误代码
provider_user_id = str(
    user_json.get("id", user_json.get("sub", user_json.get("openid", ""))
)  # 括号不匹配
```

**影响**: 
- M8 API Gateway 完全无法启动
- 阻塞所有 API 测试
- 阻塞 M5、M6 等下游模块测试

**建议修复**:
```python
# 修复方案
provider_user_id = str(
    user_json.get("id") or 
    user_json.get("sub") or 
    user_json.get("openid") or ""
)
```

#### 问题 #2: contracts 模块导入失败
**错误**: `ModuleNotFoundError: No module named 'contracts'`  
**影响**: 所有依赖 contracts 包的模块无法正常启动  
**临时方案**: sys.path manipulation (已验证可行)  
**建议修复**: 
1. 修复 contracts 包的安装配置
2. 在各模块的 pyproject.toml 中正确声明依赖
3. 或使用相对导入替代绝对导入

#### 问题 #3: 端口冲突
**冲突端口**: 8000  
**占用进程**: Manager.exe (PID 7756, Services 会话)  
**影响**: 无法在默认端口启动 M8 服务  
**建议修复**: 
1. 更改 M8 默认端口配置
2. 或提供端口自动检测和切换功能

### 🟡 P1 - 重要问题 (影响功能)

#### 问题 #4: Meilisearch 服务未启动
**影响**: 全文搜索功能不可用

#### 问题 #5: 缺少端到端测试脚本

**建议**: 创建 E2E 测试脚本验证 M1→M2→M3→M4→M5→M8→M6
