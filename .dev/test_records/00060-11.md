### 设计决策
- CLI 使用 argparse subcommands 架构（`convert` 子命令），方便未来扩展 `serve`、`info`、`batch` 等子命令
- converter 模块采用延迟导入（lazy import）避免 --help 时触发 docling/torch 的慢速导入
- 文件存在性校验在导入手动导入前进行，确保类型路径错误得到即时报错
- Web Server 使用 FastAPI 工厂函数模式（create_app()），方便测试和配置注入
- FastAPI 导入采用 try/except guard，允许仅安装 CLI 的环境不安装 FastAPI
